R のダウンロードとインストール方法 初心者向け

このページは、統計解析ソフト R をパソコンに入れるための手順を、OS別（Windows / macOS / Linux）にまとめたものです。

1. R とは？

R は、統計解析やデータ可視化に広く使われている無料のソフトウェア（オープンソース）です。R 本体をインストールすると、コマンドを入力して解析できるようになります。

ポイント：R 本体（言語＋実行環境）と、便利な統合開発環境である RStudio（任意）は別物です。

2. インストール前の確認

  • 管理者権限（Windows の場合はインストール時に必要になることがあります）
  • 空き容量（数百MB程度。パッケージを多く入れると増えます）
  • 古い R の有無：既に R が入っている場合は、最新版へ更新することをおすすめします

3. ダウンロード（CRAN から）

R は CRAN（Comprehensive R Archive Network）という公式の配布サイトから入手できます。

  1. CRAN のトップへアクセス：https://cran.r-project.org/
  2. 使っている OS（Windows / macOS / Linux）を選びます
  3. 表示される案内に従ってインストーラ（またはパッケージ）を入手します

大学や組織のネットワーク環境によっては、ダウンロードがブロックされることがあります。その場合は別の回線や、学内の情報担当部署に相談してください。

4. Windows の手順

  1. CRAN で Download R for Windows をクリック
  2. base をクリック
  3. Download R x.y.z for Windows（最新版）をクリックしてインストーラ（.exe）を保存
  4. 保存した .exe をダブルクリックして起動
  5. 基本的には「次へ」を押して進めてOK（インストール先は標準のままで問題ありません）
  6. 完了後、「スタート」メニューから R を起動できます

研究用途で多くのパッケージを入れる場合、Windows では後述の RTools が必要になることがあります（トラブルの節を参照）。

5. macOS の手順

  1. CRAN で Download R for macOS をクリック
  2. お使いの macOS バージョンに合った .pkg（インストーラ）を選び、ダウンロード
  3. .pkg を開き、画面の案内に従ってインストール
  4. インストール後、アプリケーションから R を起動できます

macOS では、パッケージのコンパイル等で Command Line Tools が必要になる場合があります（トラブルの節を参照）。

6. Linux の手順

Linux はディストリビューションによって手順が異なります。一般にはパッケージ管理（apt/dnf など）を使います。

例：Ubuntu / Debian 系

sudo apt update
sudo apt install r-base

例：Fedora / RHEL 系

sudo dnf install R

最新の R を使いたい場合は、CRAN の Linux 用案内（お使いのディストリビューションに対応）に従ってリポジトリを追加します。

7. インストール確認（動作テスト）

R を起動したら、次を入力してバージョンが表示されるか確認します。

version

簡単な計算も試してみましょう。

1 + 1
plot(1:10)

8. （任意）RStudio の導入

R の操作をしやすくするために、多くの人が RStudio（統合開発環境）を併用します。

  1. R 本体を先にインストール
  2. RStudio（Desktop 版）をダウンロードしてインストール
  3. RStudio を起動すると、内部で R を使って解析できます

RStudio は R 本体が無いと動きません（必ず R を先に入れます）。

9. よくあるトラブル

Q1. パッケージのインストールでエラーが出る（Windows）

  • ソースからのインストールが必要なパッケージでは RTools が必要なことがあります。
  • エラー文に「Rtools」や「make」「gcc」などが出ていないか確認してください。

Q2. パッケージのインストールでエラーが出る（macOS）

  • 「xcode」や「clang」などが出る場合、Command Line Tools が必要な可能性があります。
  • ターミナルで以下を実行して案内に従います：
xcode-select --install

Q3. ダウンロードページが開けない / 会社・大学のネットワークで落ちる

  • 別の回線（スマホのテザリング等）で試す
  • 組織のプロキシ設定がある場合は、ブラウザや OS の設定を確認
  • 情報担当部署へ相談

Q4. 既に古い R が入っている。上書きしてよい？

  • 基本的には最新版をインストールして問題ありません（共存する場合もあります）。
  • 更新後は、追加パッケージの再インストールが必要になることがあります。

困ったときは、エラーメッセージ（全文）と OS 情報（Windows/macOS/Linux、バージョン）を添えて相談すると解決が早いです。
